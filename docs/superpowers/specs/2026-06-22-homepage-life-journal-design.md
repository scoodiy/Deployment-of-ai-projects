# 首页「生活日志」重构设计

## 目标

将 `ayuu.fun` 首页从功能组件拼接式布局调整为克制、温暖的生活日志首页。页面让文章、日常记录、影像与音乐自然并列，避免单一内容占据全部注意力。

## 范围

- 仅调整首页布局、视觉层级和相关展示组件样式。
- 保留现有导航、搜索、公告、文章、说说、相册、播放器、歌词栏、主题切换和站点数据面板。
- 不调整数据库结构、后台管理能力、内容文件格式、路由或 Nginx/PM2 配置。

## 信息架构

1. 顶部保留现有导航与搜索入口；公告存在时在内容区顶部显示。
2. 首屏使用 12 列响应式网格：左侧为个人状态与统计，中间为最新日记/说说摘要，右侧为最新相册的代表照片。
3. 第二屏以最新文章为主体，采用固定高度、可扫描的文章卡片；播放器缩为同级的紧凑模块，歌词栏保持在播放器邻近位置。
4. 照片墙入口、说说列表、主题控制和站点数据面板在后续内容区依次出现，均保持明确的跳转入口。
5. 小屏幕按“状态、日记、照片、文章、播放器、其余模块”的顺序改为单列，不隐藏内容。

## 视觉规则

- 基调为暖白背景、深灰文字、低饱和鼠尾草绿点缀；强调色只用于状态、链接和播放器进度。
- 容器圆角收敛至约 8px，边框和阴影轻量，避免玻璃拟态、大面积渐变和悬浮感过强的卡片。
- 影像使用现有相册封面，文章与说说继续使用当前数据源，不引入外部内容服务。
- 标题、正文、元数据的字号层级清晰；卡片高度与图片比例固定，避免内容加载后布局跳动。

## 组件与数据

- `app/page.tsx` 继续读取 `getPublishedPosts()`、本地 `chatters` Markdown 与 `albums` 数据。
- 新增的首页编排组件仅接收现有文章、说说、相册和统计数据；不在渲染期间发起网络请求。
- 现有 `ProfileCard`、`CloudPlayer`、`LyricBar`、`LatestPostsCarousel`、`LatestChatterCarousel`、`ThemeToggleBlock` 与 `SiteDashboard` 可按职责复用或轻量调整；不改变其对外功能。

## 异常与空状态

- 无文章、无说说或无相册时，沿用现有兜底对象与默认封面，首页仍完整可用。
- 图片加载失败时保持容器尺寸，并显示已有的背景与文字信息，避免首屏空白。
- 所有链接继续使用既有路由，无法读取某类内容不影响其它模块展示。

## 验证

- 运行项目现有 lint、类型检查和生产构建。
- 在桌面与移动宽度检查首页：无横向溢出、无文字重叠、首屏关键信息可见。
- 通过本机 `curl` 和浏览器访问确认 `https://ayuu.fun` 返回正常页面；重启 PM2 后再次确认服务存活。

## 非目标

- 本次不改写文章内容、不迁移图片、不新增社交功能，也不改变播放器的播放逻辑。
