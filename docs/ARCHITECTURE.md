# 系统架构

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                      Frontend (React)                       │
│              Dashboard | Trading | Strategies | Risk         │
└──────────────────────────┬──────────────────────────────────┘
                           │ HTTP/WS
┌──────────────────────────┴──────────────────────────────────┐
│                    Backend (FastAPI)                         │
│         Users | Trades | Strategies | Risk | Stocks | Crypto │
└──────┬───────────┬───────────┬──────────┬───────────────────┘
       │           │           │          │
┌──────┴──┐  ┌─────┴────┐  ┌──┴──────┐  ┌┴──────────┐
│Strategy │  │  Risk    │  │  Data   │  │  QA Bot   │
│ Engine  │  │  Engine  │  │Collector│  │  (RAG)    │
└─────────┘  └──────────┘  └─────────┘  └───────────┘
       │           │           │
┌──────┴───────────┴───────────┴──────────────────────────────┐
│                    消息队列 (Kafka)                          │
└──────────────────────────┬──────────────────────────────────┘
                           │
┌──────────────────────────┴──────────────────────────────────┐
│              PostgreSQL  │  Redis  │  文件存储               │
└─────────────────────────────────────────────────────────────┘
```

## 数据流

1. **数据采集** → Data Collector 从外部API获取市场数据 → 清洗 → 存储到Redis/DB
2. **策略执行** → Strategy Engine 读取数据 → 计算信号 → 生成交易信号
3. **风控检查** → Risk Engine 检查信号 → 通过则执行交易
4. **订单执行** → Backend 处理订单 → 更新持仓和盈亏
5. **前端展示** → Frontend 通过API/WS获取实时数据并展示
6. **智能问答** → 用户提问 → QA Bot 检索知识库 → 生成回答

## 服务间通信

- **同步通信**：HTTP REST API（FastAPI）
- **异步通信**：Kafka 消息队列
- **缓存**：Redis（实时数据、会话、缓存）
- **持久化**：PostgreSQL（用户、交易记录、策略配置）
